# Contributing

## Branching
Do not push directly to `main`.
Create a feature branch and open a pull request.

## Pull Requests
Keep PRs focused and scoped to one workstream where possible.
Ask the appropriate reviewer based on the folder or feature changed.

## Security
Never commit secrets, API keys, connection strings, or `.env` files.

## MVP Rule
Prioritize MVP functionality over polish.
Do not introduce features outside the current hackathon scope without team approval.

## Team Alignment
Architecture and governance changes must stay aligned with:
- `/docs/architecture.md`
- `/docs/compliance-framework.md`

## Recommended Commit Style
- feat:
- fix:
- docs:
- refactor:
- chore:
