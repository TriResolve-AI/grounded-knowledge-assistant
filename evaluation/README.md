## Trust Score

From section 5 of the architecture document :

T = wR + wC + wP + wF

- **R** = retrieval quality signal (relevance and coverage)
- **C** = citation completeness and claim-evidence alignment
- **P** = policy/compliance pass strength
- **F** = freshness and version confidence
- Weights are configurable with sum = 1

## Risk Score and Decision Outcomes

From section 6 of the compliance framework :

Risk = f(V, U, S, E)

- **V** = violation severity
- **U** = uncertainty signal
- **S** = sensitivity classification
- **E** = evidence completeness gap

From section 5 of the compliance framework, decision outcomes are :

- **Allow** : response meets grounding and policy requirements
- **Redact** : response is partially allowed with sensitive content removed
- **Defer** : insufficient confidence or elevated risk requires human review
- **Block** : policy violation or unacceptable risk prevents release

The compliance framework provides example thresholds to be calibrated through evaluation :

- Risk < 0.3 → Allow
- 0.3 ≤ Risk < 0.6 → Redact or cautionary response
- Risk ≥ 0.6 → Defer or Block

## Evaluation Method
For each response generated by the system, we verify the five success criteria :

**Accurate responses** — Is the response grounded in the ingested documents ? Verified against ground truth from the official datasets. Unsupported claim flags tracked per architecture section 3.4.

**Strong citations** — Is a citation present and does it align with the source evidence ? Measured via C (citation completeness and claim-evidence alignment) in the Trust Score. Citation completeness checks are defined in compliance framework section 3.2. *How this is measured in practice to be confirmed with Esthefany.*

**Reduced hallucination risk** — Are unsupported claims detected and flagged per architecture section 3.4 ? *Whether the Risk Score is exposed per response to be confirmed with Esthefany.*

**Governance layer works** — Does the system correctly produce Allow / Redact / Defer / Block decisions with compliance decision rationale, as defined in compliance framework section 5 ?

**Audit trail complete** — Are all responses logged with the following fields defined in architecture section 9 : model version, prompt template version, retrieval index version, policy bundle version, and evaluation configuration version ?
